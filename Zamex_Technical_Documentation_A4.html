1. Executive Summary

Zamex нь БНХАУ-аас Монгол Улс руу хийгдэх карго тээвэрлэлтийг дижитал хэлбэрт шилжүүлсэн, олон талт ложистикийн платформ юм. Энэхүү систем нь тээвэрлэгч компаниуд (Partners), хэрэглэгчид (Clients), болон системийн админ (Super Admin) гэсэн гурван үндсэн оролцогч талыг нэгтгэдэг.

Системийн гол зорилго нь тээвэрлэлтийн ил тод байдлыг хангах, төлбөр тооцоог автоматжуулах (E-Barimt), болон тээвэрлэгч компаниудын хариуцлагыг нэмэгдүүлэх (Penalty System) юм.

PostgreSQL / SQLite Redis (Caching) Vercel / AWS

Өгөгдлийн сан нь бүрэн нормальчлагдсан (Normalized) бүтэцтэй бөгөөд гүйлгээний аюулгүй байдлыг (ACID compliance) хангадаг.

3. Core Business Modules

3.1 Logistics & Tracking Engine

Ачаа тээврийн үндсэн урсгал нь "Багц" (Batch) болон "Илгээмж" (Package) гэсэн хоёр үндсэн нэгж дээр суурилдаг.

Order Created ➔ Warehouse Receive (CN) ➔ Loading (Batching) ➔ Transport (Transit) ➔ Arrival (UB) ➔ Distribution

Key Features:

  • Barcode Generation: Илгээмж бүр дахин давтагдашгүй QR/Barcode-той байна.
  • Real-time Status: Хэрэглэгч ачаагаа хаана явж байгааг цаг тутам хянах боломжтой.
  • Batch Management: Олон мянган жижиг илгээмжийг нэг "Batch"-д нэгтгэн тээвэрлэдэг.

3.2 Marketplace & Bidding

Тээвэр зуучийн компаниуд болон захиалагчдыг холбох биржийн систем.

Component Functionality
Job Listings Хэрэглэгч ачааны мэдээлэл (жин, төрөл, чиглэл) оруулж зар тавина.
Bidding System Компаниуд үнийн санал (Bid) өгнө. Санал нь үнэ болон тээвэрлэх хугацааг агуулна.
Selection Хэрэглэгч үнэлгээ, үнэ, хурд зэрэгт үндэслэн компани сонгоно.

4. Governance & Financial Model

4.1 Penalty System (3-Strike Rule)

Чанарын хяналтыг сайжруулах зорилгоор системд "Зөрчлийн оноо"-ны механизмыг нэвтрүүлсэн.

Logic Flow

  1. Violation Report: Админ зөрчлийг (ачаа дутсан, гомдол) бүртгэнэ.
  2. Strike Increment: Компанийн "Strike Count" нэгээр нэмэгдэнэ.
  3. Automatic Trigger: Хэрэв Strike Count == 3 болвол:
    • System Status -> SUSPENDED
    • IsActive -> FALSE
    • Marketplace Access -> REVOKED

Админ зөрчлийг шийдвэрлэсний (Resolve) дараа л компанийн эрх сэргэх боломжтой.

4.2 Financial Settlement & Tax

Төлбөр тооцоо нь "Wallet" системээр дамжиж хийгддэг бөгөөд татварын хуулийн дагуу НӨАТ (VAT) автоматаар тооцогдоно.

Commission Model Example:

Description Rate / Value Calculation
Total Shipping Fee 100.0% 100,000₮ (Example)
VAT (НӨАТ) 10.0% - 9,091₮ (Sent to E-Barimt)
City Tax (НХАТ) 1.0% - 909₮
Platform Commission 5.0% - 5,000₮ (Zamex Revenue)
Net Payout ~84% 85,000₮ (To Partner Wallet)
